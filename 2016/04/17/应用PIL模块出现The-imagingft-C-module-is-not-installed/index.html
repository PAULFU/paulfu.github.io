应用PIL模块出现The _imagingft C module is not installed

发表于 2016-04-17   |   分类于 技术   |  

今天学习了廖雪峰的Python教程，其中有一个应用Python的第三方口模块库PIL的例子，用于产生验证码，觉得挺好玩的。于是动手去尝试尝试。

由于是Windows系统，下载PIL可以直接去官网下载exe文件，然后点击安装就行了。后面把示例代码贴上去运行，报错了，“The _imagingft C module is not installed”，用google搜索了一下，发现好多人都在建议去这个网站看看，这个网站内容太多，好在chrome有查找功能，直接ctrl+F，搜到一个pilolow，后面跟着一行注释，“a replacement for PIL, the Python Image Library, which provides image processing functionality and supports many file formats.
Use from PIL import Image instead of import Image”，意思很清楚，就是pillow是PIL的一个替代品，导入的时候用“from PIL import Image”语句。

但是下面给的是.whl格式的文件，这个文件不像exe文件那样点一点就安装了，这个需要用pip来install，这个网站专门介绍pip的使用姿势。执行pip install 包名.whl

我并不是很懂这个whl文件，我是用另外的方法装的。安装必须要先将之前的PIL包卸载掉，执行命令“pip uninstall pil”即可卸载。然后，直接在Windows命令行窗口输入“pip install pillow”，回车，然后稍等片刻，就帮你下载下来了。然后编写Python代码，这里一定要记得用“from PIL import Image”语句来导入模块。执行廖雪峰给的示例代码还有一个字体问题，替换成“font = ImageFont.truetype(‘C:\Windows\Fonts\Arial.ttf’, 24)”就可以了。
