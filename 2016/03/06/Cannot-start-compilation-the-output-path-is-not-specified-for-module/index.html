Cannot start compilation:the output path is not specified for module

发表于 2016-03-06   |   分类于 技术   |  

之前写Java程序用的IDE都是eclipse，因为在intellij中不知道怎么run我的程序ORZ…但是eclipse不如intellij好用，而且颜值不如intellij高（这是个看脸的时代…),今天又仔细研究了一下，发现intellij跟不一样需要不能直接run,而且在点击“edit run configuration”后又找不到”java application”这个选项，所以导致我一直不知道怎么使用intellij跑java程序。

首先你的程序必须要有main()方法
必须要有main()方法
必须要有main()方法
重要的事情说三遍。

正确地使用方法是，右键your.java文件，在弹出菜单中选择“compile your.java”,这是有可能提示cannot compile，原因是你没有为编译结果选择保存地址，正如题目那样，我google了一下，查到jetbrains官方给的答案（google搜索就是屌啊…给你最想要的），如图：

（链接在这里：jetbrains官网）
根据指示，打开project structure界面，在左侧选择project，在右侧可以看到 Project compiler output下面个文本框，这里就是配置编译结果的保存路径，可以手动输入也可导航到你想保存的任何路径，建议的路径是“你的project路径+out”，这样可以和你的项目一起管理。

下图是eclipse的文件结构：

下图是intellij的文件结构：

所以问题来了：为什么eclipse不需要手动编译直接点run按钮就可运行程序而intellij不行呢？我也不知道。。。

既然说到intellij，那么就在这里补充一个小问题的解决办法。有时我们鼠标右键想创建一个java文件，却惊讶的发现没有这个选项，这是因为你没有设定源文件，你可以先建立一个文件夹（比如名为java)来放这些.java文件,建立文件夹后，在这个文件夹上鼠标右键选择“Mark Directory as”,然后“Source Root”，搞定！
